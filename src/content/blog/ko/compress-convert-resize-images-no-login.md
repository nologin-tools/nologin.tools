이미지는 평균적인 웹 페이지 용량의 약 절반을 차지해요. JavaScript도, CSS도 아닌——이미지예요. 그런데도 사진을 압축하거나, PNG를 WebP로 변환하거나, 특정 플랫폼에 맞게 리사이즈해야 할 때 대부분의 사람들이 찾는 도구들은 슬라이더 하나 건드리기 전에 계정을 요구해요.

5초면 끝날 작업에 너무 비싼 대가예요.

브라우저 안에서 완전히 작동하고, 서버에 업로드하지 않고 클라이언트 측에서 파일을 처리하며, 탭을 여는 것 외에 아무것도 요구하지 않는 이미지 도구 카테고리가 있어요. Google이 만든 것도 있고, 월간 수백만 사용자를 보유한 개인 개발자 프로젝트도 있어요. 모두 무료이고 가입 없이 작동해요.

## 형식 선택이 생각보다 중요한 이유

대부분의 사람들은 형식을 고민하지 않고 이미지를 압축해요. JPEG가 있으면 작게 만들고, 끝이에요. 하지만 실제로는 형식 선택이 압축 설정 자체보다 더 중요한 경우가 많아요.

Google이 개발한 [WebP](https://developers.google.com/speed/webp)는 동등한 시각적 품질에서 JPEG보다 약 25~34% 작은 파일을 생성해요. Netflix와 Open Media Alliance가 지원하는 더 새로운 형식 AVIF는 JPEG 대비 파일 크기를 최대 50%까지 줄일 수 있어요. 두 형식 모두 현재 모든 주요 브라우저에서 지원돼요. 웹사이트를 위해 이미지를 최적화한다면, JPEG에서 WebP로 전환하는 것만으로도 품질 슬라이더 조작 전에 페이로드를 1/3 줄일 수 있어요.

이게 바로 압축과 형식 변환을 함께 처리하는 도구가 단순히 기존 JPEG를 압축하는 도구보다 더 유용한 이유예요. 대부분의 웹 작업에서 답은 단순해요: WebP로 변환하고, 약 80% 품질로 압축하면 끝이에요. 두 가지 로그인 없는 도구가 이걸 매우 쉽게 만들어줘요.

## Squoosh: 처음으로 사용해 볼 도구

출력을 진짜로 제어하면서 이미지를 압축해야 할 때, [Squoosh](https://squoosh.app)는 이 분야에서 가장 강력한 옵션이에요. 완전히 WebAssembly에서 실행되는 Google이 만든 오픈소스 도구로——파일이 브라우저를 절대 떠나지 않아요.

작업 흐름은 좌우 비교 뷰예요: 왼쪽에 원본, 오른쪽에 압축 미리보기. 출력 형식(MozJPEG, WebP, AVIF, JPEG XL, PNG, OxiPNG 등)을 선택하고, 품질 슬라이더를 드래그하면 실시간으로 크기 예상치가 업데이트돼요. 차이 표시는 원본의 백분율로 얼마나 킬로바이트를 절약하는지 정확히 보여줘요.

Squoosh가 대부분의 대안보다 나은 건 과도하게 단순화하지 않는다는 점이에요. 크로마 서브샘플링 조정, 인코딩 속도 선택, 고급 코덱 설정 조정이 가능해요(그게 뭔지 알고 있다면). 또는 그걸 다 무시하고 품질 슬라이더만 움직일 수도 있어요. 어느 쪽이든 확정 전에 라이브 미리보기를 볼 수 있어요——로그인 없는 도구에서는 드문 기능이에요.

리사이즈도 처리해요: 선택적 가로세로 비율 잠금 기능이 있는 너비와 높이, 여러 스케일링 알고리즘(선명도를 위한 Lanczos, 엣지에서 약간 링잉이 있는 사진에는 Mitchell). [nologin.tools의 Squoosh 목록](/tool/squoosh-app)에서 전체 기능 목록을 확인할 수 있어요. 한 가지 제한사항: 한 번에 하나씩만 처리하므로, 50장짜리 폴더가 있다면 번거로울 수 있어요.

## TinyPNG: 빠르고 완전히 자동화

[TinyPNG](https://tinypng.com)는 배치 문제를 해결해요. 한 번에 최대 20개의 이미지를 드롭하면(각 최대 5MB, 계정 불필요) 대부분의 사람들이 눈치채지 못하는 방식으로 색상 팔레트를 선택적으로 줄이는 손실 압축 최적화로 압축해요. PNG 파일은 일반적으로 60~80% 줄어들어요. JPEG와 WebP도 지원해요.

경험은 드래그앤드롭 상자, 진행 바, 다운로드 링크로 끝이에요. 설정할 게 없어요. 이건 설계상의 선택——알고리즘이 결정을 내리고, 그 결과가 충분히 좋아서 아마 의심하지 않을 거예요.

배치 처리에 가입이 필요한 도구들과 달리, TinyPNG의 20파일 제한은 배치당이지 하루당이 아니에요. 첫 번째 배치가 끝나면 다시 20개를 드래그하면 돼요. 업로드 전 갤러리를 준비하는 사진작가나 배포 전 이미지 에셋을 정리하는 개발자들에게 이 작업 흐름은 계정 없이도 충분히 감당해줘요.

브라우저 버전이 사용하는 같은 압축 품질을 개발자 API와 WordPress 플러그인으로도 이용할 수 있지만, 웹 버전이 무료 로그인 없는 옵션이에요. [nologin.tools의 TinyPNG 프로필](/tool/tinypng-com)에서 무료 티어와 유료 플랜에 포함된 내용을 확인할 수 있어요.

## ezGIF: 이름이 실력을 과소평가하는 도구

이름이 부족해요. [ezGIF](https://ezgif.com)는 계정 없이 사용할 수 있는 가장 기능이 풍부한 이미지 도구 중 하나이며, 애니메이션 GIF를 훨씬 넘어서요.

GIF 작업: 동영상 파일이나 이미지 시퀀스에서 만들기, 프레임 타이밍 최적화, 색상 줄이기, 자르기, 리사이즈, 반전, 텍스트 추가. GIF 최적화기는 특히 유용해요——애니메이션 GIF는 크기로 악명 높은데, ezGIF의 최적화는 눈에 띄는 품질 손실 없이 일반적으로 30~40%를 줄여줘요.

도구 목록은 GIF를 훨씬 넘어요. JPG/PNG/WebP 최적화기, 백분율 기반 또는 픽셀 특정 크기 조절기, 형식 변환기(AVIF, HEIC, BMP, TIFF, 많은 '모던' 대안이 슬쩍 건너뛰는 다른 형식 지원), 이미지-PDF 변환기, CSS 스프라이트 작업을 위한 스프라이트 시트 커터가 있어요.

인터페이스는 실용적——상단에 탭, 업로드 상자, 아래에 결과. 2014년경부터 재설계되지 않았고 그게 보여요. 하지만 모든 기능이 작동하며, 파일 타입 지원이 드문드문 광범위해요: iPhone의 HEIC, 스캐너의 TIFF, 최신 카메라의 AVIF. 형식이 존재하면 ezGIF가 처리할 가능성이 높아요. 전체 프로필은 [nologin.tools의 ezGIF](/tool/ezgif-com)에서 볼 수 있어요.

## Convertio와 SVGOMG: 엣지 케이스

일부 형식 변환은 너무 특수해서 대부분의 도구가 지원하지 않아요. [Convertio](https://convertio.co)는 이미지, 문서, 오디오, 동영상에 걸쳐 300개 이상의 파일 형식을 커버해요. 이미지 작업 특히: RAW 카메라 형식(CR2, NEF, ARW), DDS(게임 텍스처), TGA, EXR(VFX 파이프라인의 HDR 형식), 그리고 전문 도구들이 건너뛰는 다른 것들을 처리해요.

가입 없는 무료 변환은 합리적인 일일 제한과 파일당 100MB로 제한돼요. 파일은 Convertio의 서버에 업로드됩니다(Squoosh의 클라이언트 측 처리와 달리), 그래서 민감한 이미지의 경우 진행하기 전에 [프라이버시 정책](https://convertio.co/privacy)을 확인하세요. 제품 목업을 RAW에서 JPG로 또는 아이콘을 SVG에서 ICO로 변환하는 건 괜찮아요. 기밀적인 것에는 Squoosh의 로컬 처리가 더 안전한 선택이에요. [nologin.tools의 Convertio](/tool/convertio-co)를 확인하세요.

SVG 파일은 완전히 다른 문제예요. Figma나 Adobe Illustrator 같은 디자인 도구는 에디터 메타데이터, 중복 그룹 요소, 소수점 8자리 정밀도 숫자, 속성이 될 수 있는 인라인 스타일로 가득 찬 SVG를 내보내요. [SVGOMG](https://jakearchibald.github.io/svgomg/)는 SVGO의 브라우저 기반 프론트엔드예요——SVG를 드롭하면 라이브 미리보기를 보여주면서 노이즈를 제거해요. Figma 내보내기의 일반적인 절약은 40~70%예요. 토글 패널로 특정 최적화가 특정 SVG 트릭을 망가뜨리는 경우 비활성화할 수 있어요. 업로드 없음, 계정 없음, 모두 로컬이에요. [nologin.tools의 SVGOMG](/tool/jakearchibald-github-io-svgomg)에 프로필이 있어요.

## 작업별 도구 선택

이 도구들은 서로 경쟁하는 게 아니라 다른 필요를 커버하기 때문에 솔직한 가이드를 드릴게요:

| 작업 | 최선의 도구 | 파일이 브라우저를 떠나나요? |
|------|-----------|------------------------|
| 형식 제어로 이미지 한 장 압축 | Squoosh | 아니요 |
| PNG/JPEG/WebP 배치 압축 | TinyPNG | 예(서버 측) |
| GIF 생성 또는 최적화 | ezGIF | 예(서버 측) |
| 알고리즘 옵션으로 리사이즈 | Squoosh 또는 ezGIF | 아니요 / 예 |
| 특수 형식 변환(RAW, DDS, EXR) | Convertio | 예(서버 측) |
| 디자인 도구 SVG 최적화 | SVGOMG | 아니요 |

"파일이 브라우저를 떠나나요?" 컬럼은 프라이버시에 중요해요. Squoosh와 SVGOMG는 파일을 어디에도 보내지 않아요——모든 것이 탭 안의 WebAssembly 또는 JavaScript에서 일어나요. TinyPNG, ezGIF, Convertio는 서버에 업로드하고 처리한 후 결과를 반환해요. 세 곳 모두 짧은 보존 기간(일반적으로 24시간 이하)을 명시하고 있지만, 그들의 정책을 신뢰하는 거예요.

대부분의 일상적인 작업——제품 사진 압축, 헤더 이미지 리사이즈, JPEG를 WebP로 변환——에서는 서버 측 처리가 그 도구들이 제공하는 편의성과 형식 지원을 위한 합리적인 트레이드오프예요. 의료 이미지, 법적 문서, 낯선 서버에 두고 싶지 않은 개인적인 것: Squoosh를 쓰세요.

> 잠그지 않고도 살아남는 도구들은 실제로 좋은 경향이 있어요. Squoosh는 수익화 계획이 있는 제품으로서가 아니라 현대적인 이미지 코덱의 참조 구현으로서 Google Chrome 팀이 구축하고 유지관리해요. 이 인센티브 일치——"웹을 위한 이미지 압축을 좋게 만들기"——는 "사용자가 계정을 만들게 하기"보다 더 나은 도구를 만들어요.

[HTTP Archive의 Web Almanac](https://almanac.httparchive.org)에 따르면 웹에서의 WebP 도입이 크게 성장했지만 JPEG와 PNG가 여전히 지배적이에요. 모던 브라우저가 지원하는 것과 대부분의 사이트가 실제로 제공하는 것 사이의 격차는 테이블에 남겨진 실제 성능을 나타내요. 그 격차를 좁히는 도구 중 어느 것도 먼저 이메일을 넘겨달라고 요구하지 않아요.

더 넓은 [nologin.tools 디렉터리](/tool/nologin-tools)는 수십 개의 카테고리에 걸쳐 검증된 프라이버시 친화적 도구들을 정리하고 있어요. 이미지 도구 섹션은 더 완성도 높은 코너 중 하나예요——'가입 없음'이 타협이 아닌 기능임을 깨달은 서비스가 부족하지 않고, 컬렉션은 계속 성장하고 있어요.

솔직히 말하면, Canva 무료 플랜은 진짜 잘 만들어져 있어요. 그래서 이 글을 쓰기가 좀 어색하기도 해요. Canva는 세련되고, 빠르고, 디자인 비전문가도 그럴듯한 결과물을 낼 수 있는 수천 개의 템플릿이 있거든요. 하지만 계정이 필요해요. 그리고 그 계정 요구사항은 보이는 것보다 훨씬 많은 걸 하고 있어요.

가입하는 순간 Canva는 당신의 이메일, 디자인 기록, IP 주소, 사용 패턴을 얻고, 월 $15 구독을 팔 직통 채널을 확보해요. 이게 거래의 내용이에요. 당신의 관심과 데이터가 무료 서비스를 떠받치는 거죠. 대부분의 사람들은 별 생각 없이 이 거래를 받아들여요. 괜찮을 수도 있어요. 하지만 대안이 존재한다는 걸 알 필요는 있어요. 회원가입 없이 완전히 작동하는 브라우저 기반 디자인 도구들이 놀라울 만큼 강력해졌거든요.

Canva의 로그인 요구는 예상치 못한 상황에서도 걸림돌이 돼요. 공용 사무실 컴퓨터, 클라이언트 노트북, 평소에 안 쓰는 기기에서의 급한 작업, 또는 기업 SaaS 벤더가 디자인 내용을 기록하길 원치 않는 민감한 업무 같은 상황들이요. 계정 난립 문제도 있어요. 수십 개의 웹 도구를 쓰고 각각 회원가입이 필요하다면, 결국 수십 개의 마케팅 이메일 구독을 해지해야 하고, 수십 곳에 이메일 주소가 남아요.

아래 다섯 가지 로그인 불필요 도구들은 탭을 열고, 만들고, 내보내고, 탭을 닫으면 끝이에요. 이메일 인증도, 온보딩 체크리스트도, "이번 달 무료 내보내기 5회 중 3회 사용했어요" 같은 경고도 없어요.

## Photopea: Photoshop보다 더 비슷할 줄 몰랐어요

[Photopea](https://www.photopea.com)는 이 목록에서 기술적으로 가장 인상적인 도구이자, 가장 뜻밖의 도구예요. Ivan Kutskir라는 개발자 한 명이 거의 혼자서, PSD 파일(Photoshop 기본 형식), XCF(GIMP 형식), Sketch, AI 파일(Adobe Illustrator), PDF, SVG, 일반 웹 형식을 모두 처리하는 브라우저 기반 이미지 편집기를 만들었어요.

페이지를 열면 바로 편집기가 나와요. 이름을 묻는 로딩 화면도 없고, 플랜 선택도 없고, 건너뛸 투어도 없어요. 2010년 Photoshop처럼 생겼는데, 괜찮아요. 2010년 Photoshop은 엄청나게 기능이 풍부했거든요.

레이어, 조정 레이어, 스마트 오브젝트, 블렌드 모드, 마스크, 필터, 커브가 다 있어요. 그것들의 근사치가 아니라 실제 기능이요. Photoshop이 없는 기기에서 클라이언트의 PSD 파일을 Photopea로 열어서 텍스트 레이어와 레이어 효과가 그대로 유지된 채로 작업한 적도 있어요. Canva는 이걸 아예 못 해요. 템플릿 도구지, 파일 편집기가 아니거든요.

수익 모델은 무료 버전에 광고 (월 약 $9의 광고 없는 유료 플랜이 Kutskir의 개인 개발을 지원해요). 처리는 전부 브라우저 안에서 이루어지고, 파일은 서버로 전송되지 않아요. Photopea의 월간 활성 사용자는 약 1,000만 명이라고 하는데, 1인 프로젝트치고는 놀라운 숫자예요. [Kutskir의 개발 블로그](https://blog.photopea.com)는 이 규모를 혼자 어떻게 유지하는지 궁금하다면 읽어볼 만해요.

[nologin.tools의 Photopea 페이지](/tool/photopea-com)에서 전체 기능 요약을 확인하세요.

**이런 분께 추천**: 레이어 파일(PSD, XCF, AI) 편집, 사진 보정, 합성, 진짜 레이어 제어가 필요한 작업.
**이런 분께는 비추**: 템플릿으로 간단한 SNS 이미지를 만들고 싶을 뿐이라면. 그게 이 도구의 목적이 아니에요.

## Excalidraw: 의도적으로 거칠게 만든 다이어그램 도구

[Excalidraw](https://excalidraw.com)는 모든 것을 손으로 그린 것처럼 보이게 해요. 이건 우연이 아니에요. 약간 삐뚤어진 도형과 손글씨 폰트는 "초안"을 의미해요. 시스템 아키텍처 다이어그램, 기술 플로우차트, 아직 정리 중인 와이어프레임에 딱 맞는 신호예요.

로그인 불필요는 설계 철학에 녹아 있어요. 실시간 협업은 공유 URL로 작동하고, 참여자 누구에게도 계정이 필요 없어요. 협업 세션은 엔드투엔드 암호화되어 있어서, Excalidraw 서버조차 무엇을 그리고 있는지 볼 수 없어요. 이건 민감한 기술 작업에서 의미 있는 프라이버시 특성이에요.

Excalidraw는 [MIT 라이선스 오픈 소스](https://github.com/excalidraw/excalidraw)로 85,000개 이상의 GitHub Star를 보유하고 있어요. 현존하는 오픈 소스 드로잉 도구 중 가장 많이 별을 받은 것 중 하나예요. Notion, GitLab, Linear, 수십 개의 다른 제품에 내장되어 있어요. 라이브러리가 광범위하게 사용된다는 건, 그것에 의존하는 사람들이 유지하고 개선한다는 의미예요.

Canva와의 비교가 흥미로워요. Canva로 다듬은 다이어그램을 디자인 리뷰에 가져가면, 사람들은 최종안을 발표한다고 생각해요. Excalidraw 스케치를 가져가면 대화는 열린 채로 유지돼요. 사람들이 추가하고, 의문을 제기하고, 시작점으로 취급하는 것에 편안함을 느껴요. 두 모드 다 프로젝트의 다른 단계에서 중요해요.

[nologin.tools의 Excalidraw 전체 프로필](/tool/excalidraw-com)도 확인해 보세요.

**이런 분께 추천**: 기술 다이어그램, 시스템 아키텍처 스케치, 와이어프레임, "러프 드래프트"가 의도한 메시지인 시각적 브레인스토밍.
**이런 분께는 비추**: 인쇄되거나 게시될 정돈된 결과물이 필요한 경우.

## tldraw: 팀에게 실제로 작동하는 화이트보드

[tldraw](https://tldraw.com)는 Excalidraw와 Miro 사이 어딘가에 있어요. 결과물이 더 깔끔하고 손으로 그린 느낌이 없지만, 전통적인 디자인 도구가 아닌 무한 캔버스 화이트보드예요. 도형은 선명하고, UI는 현대적이며, 실시간 협업은 Excalidraw와 같은 방식으로 작동해요. URL을 공유하면 누구든 계정 없이 참여할 수 있어요.

tldraw가 특히 잘하는 건 공간적 시각 사고예요. 포스트잇을 붙이고, 예쁜 화살표를 그리고, 서식 있는 텍스트를 쓰고, 이미지를 삽입하고, 프레임으로 캔버스 섹션을 그룹화할 수 있어요. PNG와 SVG로 내보내기 가능. 전체 경험이 반응성 있어서 2008년에 설계된 것처럼 느껴지지 않아요.

개발자 SDK가 널리 사용되고 있어요. [tldraw의 GitHub 저장소](https://github.com/tldraw/tldraw)는 약 35,000개의 Star를 가지고 있고, `tldraw` npm 패키지는 캔버스 경험이 필요한 많은 제품에 내장되어 있어요. 이는 엔드 사용자뿐만 아니라 그 위에 제품을 구축하는 사람들이 기반 코드를 검토하고 개선한다는 의미예요.

2024년에 tldraw는 AI 보조 캔버스 기능 데모를 공개했어요. UI 와이어프레임을 스케치하면 작동하는 HTML을 생성하는 것 같은 기능이요. 이 기능들은 실험적이지만, 방향성을 보면 tldraw가 캔버스를 더 스마트하게 만드는 데 투자하고 있다는 게 분명해요. 주목할 만해요.

[nologin.tools의 tldraw 전체 목록](/tool/tldraw-com)도 확인하세요.

**이런 분께 추천**: 팀 브레인스토밍 세션, 시각적 계획 수립, 가입 없이 협업이 필요한 가벼운 와이어프레임.
**이런 분께는 비추**: 특정 파일 형식 출력이 필요한 혼자 하는 작업.

## SVG-Edit: 브라우저에서 진짜 벡터 편집

[SVG-Edit](https://svgedit.netlify.app/editor/index.html)는 이 목록에서 가장 오래된 도구이고, 그렇게 생겼어요. 인터페이스는 기능적이지만 낡았어요. 유틸리티 나이프처럼 기능적이고 낡은 것과 같아요. 예쁘게 보이려는 노력 없이 일을 해내요.

SVG-Edit를 선택하는 이유는 간단해요. 소프트웨어 설치 없이 SVG 파일을 만들거나 편집해야 하고, 깔끔한 출력물이 필요한 경우예요. SVG-Edit는 SVG 형식에 직접 써요. 내보낼 때 변환되는 내부 표현이 없어요. 그린 것이 파일에 들어가는 것이에요.

이건 웹 작업에서 중요해요. Canva는 일부 플랜에서 SVG 내보내기를 할 수 있지만, Canva의 SVG 출력은 복잡한 경향이 있어요. 중첩된 `<g>` 요소들, 자동 생성된 ID, 불필요한 속성들이 많아요. SVG-Edit의 출력은 그에 비해 간결해서 코드 편집기에서 수동으로 수정하거나 최적화 도구를 통해 처리하기 쉬워요.

오픈 소스이고 수년째 그래왔어요. [SVG-Edit GitHub 저장소](https://github.com/SVG-Edit/svgedit)는 여러 번의 대규모 리라이트를 거쳐도 유지되고 있어요. 코드베이스는 이제 모듈식 ES 모듈이에요. 가장 화려한 프로젝트는 아니지만, 적극적으로 유지 관리되고 신뢰할 수 있어요.

[nologin.tools의 SVG-Edit](/tool/svgedit-netlify-app-editor-index-html)에서 빠른 요약을 확인하세요.

**이런 분께 추천**: 코드에서 직접 사용할 깔끔한 SVG 아이콘, 로고, 일러스트가 필요한 웹 개발자.
**이런 분께는 비추**: 친근한 인터페이스가 필요하거나, 사진 작업을 하거나, 낡은 UI를 못 참는 경우.

## Haikei: Canva가 못 하는 것을 해주는 도구

[Haikei](https://haikei.app)는 매우 전문화된 도구예요. 그래서 대체 도구가 아닌 다른 도구들과 함께 사용하는 걸 권장해요. 장식적인 SVG 배경, 블롭 형태, 웨이브 구분선, 그라데이션 메쉬, 레이어드 웨이브, 추상 패턴 등 30가지 이상의 생성기 유형을 제공하고, 프로덕션 준비가 된 깔끔한 SVG를 내보내요.

워크플로는 이렇게 돼요. 생성기를 선택하고, 파라미터를 조정하고, 마음에 드는 게 나올 때까지 랜덤화를 눌러서, 내보내요. 2분이면 끝이에요. 출력물은 확장 가능한 벡터 그래픽이라 어떤 화면 크기에서도 픽셀화되지 않고, 파일이 작아서 원하면 HTML에 직접 인라인으로 쓸 수도 있어요.

Canva에도 배경 옵션이 있지만, 고정 크기의 비트맵이지 설정 가능한 벡터 자산이 아니에요. 웹사이트를 만들면서 브랜드 색상에 맞고 섹션 사이에서 자연스럽게 전환되는 웨이브 구분선이 필요하다면, Haikei가 그 제어권을 정확하게 드려요. 이건 Canva의 대체가 아니라 Canva가 남긴 공백을 채우는 도구예요.

핵심 생성기는 계정 없이 무료로 사용할 수 있어요. 더 많은 생성기 유형이 필요하면 Pro 티어에서 추가할 수 있어요.

[nologin.tools의 Haikei](/tool/haikei-app)에서 무료 티어에 포함된 내용을 확인하세요.

**이런 분께 추천**: 랜딩 페이지와 앱 인터페이스를 위한 배경, 블롭 형태, 웨이브 구분선 같은 장식적 SVG 자산을 만드는 웹 개발자와 디자이너.
**이런 분께는 비추**: 전문 배경 생성기가 아닌 범용 디자인 도구가 필요한 경우.

## 어느 도구가 나에게 맞을까요?

"Canva 대체"라는 프레임이 약간 오해를 불러일으켜요. Canva는 주로 편집기가 붙은 템플릿 라이브러리거든요. Canva를 유용하게 만드는 건 편집기 자체가 아니라, SNS 게시물, 프레젠테이션, 전단지, 명함을 위한 수천 개의 기성 템플릿이에요. 이 다섯 가지 도구는 그것을 복제하지 않아요.

대신 각자의 특정 영역에서 진짜 깊이가 있어요.

| 도구 | 주요 용도 | 오픈 소스 | 내보내기 형식 |
|------|-----------|----------|--------------|
| [Photopea](https://www.photopea.com) | 사진 편집, 레이어 파일 | 아니요 | PSD, PNG, JPG, SVG, PDF |
| [Excalidraw](https://excalidraw.com) | 기술 다이어그램, 와이어프레임 | 예 (MIT) | PNG, SVG |
| [tldraw](https://tldraw.com) | 협업 화이트보드 | 예 | PNG, SVG |
| SVG-Edit | 깔끔한 SVG 벡터 생성 | 예 | SVG |
| [Haikei](https://haikei.app) | 장식적 SVG 자산 | 아니요 | SVG, PNG |

회원가입이 필요 없어요. tldraw의 실시간 멀티플레이어 동기화를 제외하면 파일이 서버에서 처리되지 않아요 (그게 그 기능의 목적이기도 하고요). 모두 플러그인 없이 최신 브라우저에서 작동해요.

> 개인정보를 중요시한다는 건 편집증적이 된다는 게 아니에요. 어떤 데이터를 남길지 의식적으로 선택하는 거예요.

이 도구들을 함께 사용하면, Canva의 템플릿 생태계가 꼭 필요하지 않은 대부분의 디자인 작업을 커버할 수 있어요. 계정도, 디자인 기록도, 관리할 구독도 남기지 않고요. 더 넓은 디렉토리는 [nologin.tools](/tool/nologin-tools)에서 확인할 수 있어요.

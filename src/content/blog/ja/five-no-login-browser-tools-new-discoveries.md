先日、[BreezePDF](https://breezepdf.com/?v=3) が「無料のインブラウザ PDF エディタ」というキャッチコピーとともに Hacker News に投稿され、フロントページに上がりました。トップコメントで語られていたのは機能の話ではなく、「メールアドレスを入力しなくてもそのまま使えた」という一点でした。その反応が多くの共感を集めていたのが印象的です。

この反応は、2026 年における人々の期待値がどこにあるかを示しています。サインアップの壁が当たり前になりすぎて、「すぐ使えるツール」がニュースになってしまう状況は、本来は逆のはずです。登録を要求する側が例外的な選択をしているのであって、ブラウザ内ですべてを処理する——サーバーなし、アカウントなし、データ保存なし——というアプローチは、技術的にはむしろシンプルな道です。成長指標よりもユーザー体験を優先する姿勢が必要なだけで。

ここでは、そのシンプルな道を選んだ5つのツールを紹介します。

## BreezePDF：開いたらすぐ使える PDF エディタ

多くの人が PDF エディタを探すきっかけは、だいたい同じです。誰かからフォームが送られてきて、記入して返さないといけないのに、Acrobat がない。そこからオンラインツールを探し始め、アカウント登録が必要と分かって別のツールを探し、また別のを試して……気づけば5分が経っている、という流れです。

BreezePDF はその流れを断ち切ります。URL を開いて PDF をアップロードすれば、すぐに編集を始められます。フォームへの記入、注釈の追加、テキストの挿入、ページの並び替えといった、PDF エディタを開く理由の 90% をカバーする操作に対応しています。処理はすべてブラウザ内で完結するため、ファイルが外部のサーバーに送信されることはありません。

比較として挙げたいのが [PDF24 Tools](/tool/tools-pdf24-org-en) です。PDF 関連作業における信頼性の高いログイン不要ツールとして長年使われており、結合・分割・圧縮・変換に対応しています。BreezePDF は単一ドキュメントの内容編集に特化しています。両者は競合ではなく補完関係にあり、どちらも知っておく価値があります。PDF24 がスイスアーミーナイフなら、BreezePDF はメスのようなツールです。

BreezePDF が注目される理由は機能だけではありません。タイミングも重要です。[以前は無料だった複数の PDF ツールが、エクスポート機能をアカウント登録の後ろに隠すようになってきた](https://smallpdf.com/pricing)まさにそのタイミングで登場したからです。このトレンドが続く中で、登録不要を明言した新しいツールの登場は、意味のある表明だと言えます。

## Datasette Lite：ブラウザのタブで動くデータベース探索ツール

CSV や SQLite ファイルを受け取って中身を調べたいとき、選択肢はいくつかあります。Excel で開く（小さいファイルならともかく、大きいと辛い）、ローカルのデータベースにインポートする（セットアップが必要）、クラウドサービスにアップロードする（データがどこに行くか心配になる）。

[Datasette Lite](https://lite.datasette.io) はそれとは異なる選択肢です。WebAssembly を使ってブラウザ内で Datasette アプリケーション全体を動かします。具体的には Pyodide を使って Python インタープリタを WASM にコンパイルしています。ローカルファイルや URL から SQLite ファイルを読み込み、SQL クエリを実行し、データのフィルタリングや並び替え、結果のエクスポートが可能です。データはマシンの外に出ることがありません。

これは技術的に本物の成果です。数年前であれば、サーバーなしでブラウザ内に Python Web フレームワークを動かすことは現実的ではありませんでした。[WebAssembly の仕様](https://webassembly.org/docs/use-cases/)が成熟し、このような移植が実用的になってきており、Datasette Lite はその好例と言えるでしょう。

ジャーナリストや研究者、機密性の高いファイルのデータ分析を行う方にとって、プライバシーの観点は利便性と同じくらい重要です。医療記録や財務データのデータベースをクラウドサービスにアップロードしてクエリを実行するのは、リスクのある選択です。同じクエリをブラウザのタブ内でローカル実行し、データが一切外に出ないようにするのが、あるべき姿です。

Datasette Lite は URL からのデータ読み込みにも対応しているため、政府の公開データセットやオープンデータポータルをローカル環境を構築せずに探索するのにも役立ちます。

## led.run：どんな画面でも表示ボードに変換

[led.run](/tool/led-run) が活躍する場面は具体的です。会場、イベント、教室、プレゼンテーションなどで、スクロールするテキストを画面に表示したいのに、ソフトウェアのインストールも専用の表示システムのセットアップもしたくないとき。会議の歓迎メッセージ、Q&A セッションのプロンプト、カウントダウン、名前の表示ボードといった用途が考えられます。

led.run はブラウザベースの表示ツールキットで、あらゆる画面をコントロール可能なディスプレイに変換できます。テキストを入力し、フォントサイズとカラースキームを選んで、スクロール速度を設定し、ブラウザで URL を開くだけです。モダンなブラウザが動くデバイスであれば何でも使えます。URL を共有することで別のデバイスから操作することも可能です。アプリのインストール不要、アカウント不要、「複数ディスプレイ」機能のためのサブスクリプション不要です。

このツールが際立っているのは、狭い問題をうまく解決しているという点です。注目したいのはその設計思想です。登録・分析・バッジ印刷を備えた「完全な機能のイベント管理プラットフォーム」を作るのではなく、led.run はただ一つのこと——テキストを画面に表示する——を余計なものを一切つけずに実現しています。

このアプローチはログイン不要のツールの間でよく見られるようになっています。「アカウント不要」という制約は、自然とデザインをシンプルな方向へ押し進めます。ユーザーの永続的な状態に依存できないなら、それなしでも動くように作るしかない。その結果として、より良いツールが生まれることが多いのです。

## iFormat.io：メールアドレス不要のファイル形式変換

ファイルの形式変換は頻繁に発生するニーズで、多くのツールが存在しますが、そのほとんどはアカウント登録を求めます。HEIC の写真を JPEG に変換する、DOCX を PDF にする、音声を M4A ではなく MP3 で書き出す。

[iFormat.io](/tool/iformat-io) は登録不要で 500 種類以上の形式変換に対応しています。カバー範囲は広く、音声・動画・画像・ドキュメント・電子書籍・アーカイブを含みます。ファイルの処理はサーバーサイドで行われますが（すべての形式のバイナリ変換をブラウザ内で行うのはまだ現実的ではないため）、アカウントは不要で、ファイルは処理後に保存されることなく削除されます。

比較対象として [Convertio](/tool/convertio-co) があります。この分野で信頼性の高い選択肢として長く使われてきましたが、近年無料枠が厳しくなっています。アカウントなしでも変換できますが、ファイルサイズや変換回数の制限が増えました。大きなファイルや一括変換で Convertio の制限が問題になる場合、iFormat.io は知っておく価値のある代替手段です。

より単純な変換——画像のリサイズ、PNG の圧縮、画像フォーマットの変換——には、[TinyWow](/tool/tinywow-com) も登録不要で幅広い操作に対応しています。何を変換するかによって適切なツールは異なりますが、汎用的なファイル形式変換であれば iFormat.io が最も広い範囲をカバーしており、何も要求してきません。

## SiteAge：アカウント不要でどんなウェブサイトの歴史も調査

新しいツールやサービスを評価するとき、どれくらい前から存在するかは重要な情報です。6ヶ月前にオープンしたサービスと10年間運営を続けているサービスでは、信頼性の意味合いが異なります。3年間で2回も名前が変わったサービスは、一貫したアイデンティティを保ってきたサービスとは別の目で見る必要があります。年齢と継続性はシグナルです。

[SiteAge](/tool/siteage-org) は [Internet Archive の Wayback Machine](https://web.archive.org/) からこの情報を収集します。Wayback Machine は最も長く運営されているウェブ保存プロジェクトの一つです。URL を入力すると、SiteAge は最も古いアーカイブスナップショット、ドメイン登録の履歴、そしてサイトがどのように変化してきたかのタイムラインを表示します。公開情報から集められた数年分の歴史的データを、アカウントを作ることなく確認できます。

これはログイン不要ツールを検証したいまさにそのシチュエーションで役立ちます。使えそうなツールを見つけた、無料でプライバシーに配慮していると書いてある、でも信頼に足るほどの歴史があるかを確かめたい。Wayback Machine のインターフェイス自体でも調べられますが、SiteAge はより素早く重要な情報を表示します。サイトが最初に登場した時期、成長しているか縮小しているか、現在のドメインで何年運営されているか。

> 5年間アカウント登録なしで運営し続けているツールは、先月「今のところログイン不要」として立ち上がったツールとは、異なる種類の約束をしています。

より大きな視点で見ると、プライバシーに配慮したツールは商業的な製品に比べて情報が少ないことが多いです。ツールが謳っていることを実際にしており、収集すべきでないデータをこっそり集めていないと信じるしかない。そのツールが年単位で一貫して運営されてきたことを Archive で確認できることは、信頼を積み上げる一つの根拠になります。

## この5つのツールに共通すること

これら5つのツールは同じカテゴリに属してはいません。PDF 編集、データベース探索、表示システム、ファイル変換、歴史調査は、通常ひとつの特集記事に並ぶことはないでしょう。しかし構造的な共通点があります。

どれも、何か具体的なことをするために、自分の情報を差し出すことを要求しません。それぞれが異なる方向から「ユーザーはアカウントなしで今すぐ使えるべきだ」という正しい答えに到達しています。BreezePDF は PDF 編集がクライアントサイドで完結できるから。Datasette Lite は WebAssembly によって複雑なソフトウェアをブラウザ内で動かすことが可能になったから。led.run は URL ベースの状態管理で表示ツールとして十分に機能するから。SiteAge は元のデータが既にパブリックだから。

[nologin.tools のディレクトリ](/tool/nologin-tools)では、この原則に基づいて整理された 100 以上のツールを収録しています。発見の難しさは現実の問題です。ログイン不要のツールはサブスクリプション製品が使うようなマーケティングチャネルには現れません。成長予算もなく、オンボーディングシーケンスも最適化されていません。口コミとキュレーションされたリストが、人々がそれらを見つける手段です。

BreezePDF を Hacker News に投稿した例は、この仕組みがどう機能するかを示す好例です。誰かが作ったものを投稿し、反響を集めたコメントは機能の話ではなく、「すぐ使えた」という事実についてでした。ソフトウェアを職業として作っている技術的に洗練された聴衆からのその反応は、注目に値します。

このような方向でつくられるツールは増えています。ブラウザの機能は拡張し続けています。[WebAssembly の仕様](https://webassembly.org/roadmap/)はスレッド、ガベージコレクション、デバッグサポートの改善など新機能を追加し続けており、より複雑なソフトウェアをブラウザに移植することが現実的になっています。「すべてあなたのブラウザ内で動作し、サーバーは不要」と正直に言えるツールの数は増え続けるでしょう。

注目すべき組み合わせは、プライバシーへの配慮、ブラウザネイティブ、アカウント不要、の三つです。この組み合わせは本来あるべき数より少ないですが、以前より少なくなくなってきています。

この条件に当てはまるツールを見つけたのに、まだ[ディレクトリ](/tool/nologin-tools)に登録されていなければ、[申請フォーム](/submit)から約2分で登録できます。

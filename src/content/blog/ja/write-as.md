ほとんどの投稿プラットフォームは同じ手順で始まります。メールアドレスの入力、パスワードの設定、そして一度も読まない利用規約のチェックボックス。まだ一文字も書いていないのに、すでにプロファイルが作られています。

Write.as はそのすべてを省略します。サイトを開いて、書いて、公開ボタンを押す。永続的な URL が手に入ります。それだけです。

## Write.as が実際にできること

Write.as は、ひとつのシンプルな前提に基づいたミニマリストの執筆・公開プラットフォームです。あなたが書いた内容は、あなたが誰であるかよりも大切だという前提です。サイトを開くと、すぐにエディターが表示されます。クリーンな全画面のテキストエリアで、サイドバーも、ニュースレターの購読を促すポップアップもありません。

好きなことを書いて、「公開」をクリックします。すると `write.as/abc123xyz` のような一意の URL が生成され、リンクを持つ誰もがすぐにアクセスできます。アカウントを作る必要も、メールを確認する必要もありません。ただ書くだけです。

裏側では、Write.as がブラウザにローカルで「オーナートークン」を保存します。後で投稿を編集・削除できるようにするためです。複数のデバイスで恒久的に管理したい場合は、任意でアカウントを作成できます。ただし、コアな用途には一切必要ありません。

インターフェイス自体は、ほとんど攻撃的なほどシンプルです。Markdown をサポートしています。エディターには、読みやすいクリーンなテキストが表示されるだけです。ワープロのツールバーも、書式リボンも、絵文字リアクションのリボンもありません。多くのウェブエディターを視覚的に疲れると感じるライターにとって、この簡素さこそが製品の本質です。

## プライバシーファーストのアーキテクチャ

ほとんどのコンテンツプラットフォームは、あなたについてできる限り多くを知るように設計されています。どの投稿を読んだか、各投稿にどれくらい時間を費やしたか、次に何をクリックしたか。そのデータがビジネスモデルです。

Write.as は正反対の前提で設計されました。収集するデータは最小限で、広告トラッキングを行わず、公開に個人を特定できる情報は一切必要としません。これは通常の広告プラットフォームにあとからくっつけたマーケティングの主張ではありません。最初から技術的なアーキテクチャに組み込まれています。

Write.as の背後にあるオープンソースエンジンは **WriteFreely** と呼ばれ、GitHub の [github.com/writefreely/writefreely](https://github.com/writefreely/writefreely) で公開されています。これが意味するのは次のことです。

- 誰でもソフトウェアの動作を正確に確認できる
- 組織や個人が独自のインスタンスをセルフホストできる
- プラットフォームのプライバシーの約束は単なるポリシーではなく、検証可能なコード

WriteFreely のセルフホストは、ジャーナリズムの集団、学術グループ、プライバシーを重視するコミュニティの間でますます広まっています。特定企業のサーバーに依存せず、分散型・連合型の公開環境を求める人々に支持されています。

> 「私は、自分に何も求めない執筆ツールが欲しい。」——Write.as に集まるコミュニティで繰り返し聞かれる言葉です

この言葉は、なぜこのツールが一般的なブログプラットフォームでは決してうまく対応できなかった読者層を獲得できたかを端的に表しています。

## ほかのノーログイン執筆ツールとの比較

私たちのディレクトリには、サインアップなしで使える執筆ツールがいくつかありますが、それぞれ目的が異なります。Write.as の位置づけは以下のとおりです。

| ツール | 主な用途 | ログイン必要？ | ウェブへ公開？ |
|--------|----------|---------------|---------------|
| [ZenPen](/tool/zenpen-io) | 集中して下書き | 不要 | 不可（ローカルのみ） |
| [Dillinger](/tool/dillinger-io) | Markdown 編集 | 不要 | エクスポートのみ |
| [StackEdit](/tool/stackedit-io) | 同期付き Markdown | 任意 | 直接 URL なし |
| [Hemingway Editor](/tool/hemingwayapp-com) | 文体・可読性チェック | 不要 | 不可 |
| Write.as | 匿名での公開 | 不要 | 可（即時 URL） |

重要な違いがあります。上記にリストしたほかのノーログイン執筆ツールは、本質的にローカルの下書き環境です。Write.as だけが、書くことの自然な終着点がインターネット上の公開 URL であり、アカウントなしで誰でもアクセスできる場所になっています。

これにより、ほかのツールでは対応できない特定の用途に役立ちます。

- Google Docs（編集にサインインが必要）を使わずに下書きをコラボレーターと共有する
- プラットフォームアカウントに名前を紐づけずに、声明や公開書簡を発表する
- 会議や会話のために、すぐに共有できる参照文書を作成する
- 一度だけ共有して忘れたいものを書く

## ダークパターンのないフリーミアム

Write.as には有料プランがあります。カスタムドメイン、複数の名前付きブログ、チームアカウント、アナリティクスなどの機能が追加されます。長期的なオンラインプレゼンスを構築したい人にとって、これらは本当に役立つ機能です。

ただし、有料プランはあくまで、すでに無料プランで価値を得た人へのアップセルです。無料プランを意図的に不満足にしてアップグレードを強制するものではありません。コアな流れ（エディターを開く→書く→公開する→URL を得る）は完全に無料で、アカウントも一切不要です。

これは重要な違いです。多くのフリーミアムツールは、サブスクリプションへの誘導のために意図的に無料プランを使いにくくしています。Write.as の無料プランは、宣伝どおりの機能をきちんと提供します。アカウントを作らなくても、完全に機能する公開ツールを使い続けられます。

たとえば [QuillBot](/tool/quillbot-com) のような多くの AI ライティングアシスタントは、機能する無料プランを提供しながらも、常にアップグレードを促します。Write.as は、サブスクリプションの勧誘で執筆体験を中断させることはありません。

## 実際に誰が使っているのか

Write.as は、普段はあまり交わらないいくつかのコミュニティで利用者を獲得しています。

**プライバシー支持者**は、デジタルアイデンティティを構築せずに公開するために使っています。アカウント不要と最小限のデータ収集の組み合わせは、プライバシーコミュニティが掲げる価値観に本当に合致するプラットフォームのひとつです。

**ジャーナリストや内部告発者**は、匿名性が重要な場面で、WriteFreely ベースのインスタンスを通じて文書や声明を公開してきました。セルフホストオプションにより、公開は Write.as 社に依存しません。

**開発者**は、素早い技術記事の執筆、RFC スタイルの文書、または完全な CMS を用意せずに共有リンクが必要な社内アナウンスに使っています。

**アイデアを探求するライター**は、プレッシャーのない空間として利用しています。アカウントも、フォロワーも、アナリティクスも、コメントもないとき、ソーシャルな承認のフィードバックループが完全に消えます。それが解放感になるか恐怖になるかは、読者との関係次第です。

**オープンソースコミュニティ**は、コミュニティブログや集合的なジャーナルのためにセルフホストの WriteFreely インスタンスを運営しており、プラットフォームに組み込まれた ActivityPub サポートで連合フォローが可能です。

## ActivityPub と連合ウェブ

特筆すべき機能があります。Write.as は [ActivityPub](https://www.w3.org/TR/activitypub/) をサポートしています。Mastodon や Pixelfed などの分散型ソーシャルプラットフォームを支えるオープンスタンダードです。つまり、Write.as（または任意のセルフホスト WriteFreely インスタンス）でアカウントを作成すると、あなたのブログは Mastodon などの連合プラットフォームからフォローできるようになります。

読者は自分の Mastodon アカウントから Write.as のブログをフォローできます。新しい投稿は時系列でフェデレーションタイムラインに表示されます。アルゴリズムフィードもなく、エンゲージメントの最適化もありません。ただ、あなたをフォローすることを選んだ人たちに、時系列で投稿が届くだけです。

この連合設計は、より広い哲学を反映しています。あなたのコンテンツはオープンなウェブに存在すべきであり、特定プラットフォームの囲われた庭に閉じ込められるべきではないということです。

## はじめ方

セットアップガイドは一切不要です。[write.as](https://write.as) を開けば、エディターがそこにあります。書き始めてください。準備ができたら「公開」をクリックします。後で編集したい場合は、投稿の URL とオーナートークンを安全な場所に保存しておきましょう。

定期的に使うようになったら、無料アカウントの作成を検討する価値があります。すべての投稿を一箇所で管理でき、恒久的なブログ URL も手に入ります。ただ、その判断をする前に、すでに数十件の投稿をしているかもしれません。

自分の公開インフラを構築したい方には、WriteFreely リポジトリに Docker サポートとセルフホスティングのための包括的なドキュメントが含まれています。このソフトウェアは世界中のコミュニティで積極的にメンテナンスされています。

ウェブには多くの「重さ」が蓄積されています。サインアップフロー、トラッキングピクセル、Cookie の同意バナー、アルゴリズムフィード。Write.as は、ウェブへの公開が常にそれを取り巻くエコシステムが示唆するよりもシンプルであったこと——そして、重要なことを何も犠牲にせず、今もそのシンプルさを保てることを思い出させてくれます。

Canvaの無料プランについて正直に言うと、それは本当に優れています。これが書きにくい理由です。Canvaは洗練されていて、速く、デザイナーではない人でも有能に見せてくれる何千ものテンプレートを持っています。しかし、アカウントが必要です。そしてそのアカウント要件は、見た目以上に多くのことをしています。

サインアップすると、Canvaはあなたのメールアドレス、デザイン履歴、IPアドレス、使用パターン、そして月額15ドルのサブスクリプションをマーケティングするための直接チャンネルを取得します。これが取引です — あなたの注意とデータが無料ティアを支えています。ほとんどの人はそれを考えずに受け入れます。あなたもそれで構わないかもしれません。しかし、代替手段が存在することを知る価値があります。なぜなら、サインアップなしで完全に機能するブラウザベースのデザインツールが驚くほど有能になってきているからです。

Canvaのログイン要件は、予期しない特定の状況でも問題になります：共有オフィスコンピューター、クライアントのノートパソコン、普段使わないデバイスでのちょっとした作業、あるいは企業のSaaSベンダーに設計内容をログされたくない機密性の高い作業。アカウントの増殖問題もあります。十数個のウェブツールを使っていて、それぞれにサインアップが必要な場合、最終的に退会しなければならない十数個のマーケティングメールの購読、十数か所にあなたのメールアドレスが存在することになります。

これら5つのログイン不要ツールを使えば、タブを開いて、何かを作り、エクスポートして、タブを閉じるだけです。メール確認なし、オンボーディングチェックリストなし、「今月の5つの無料エクスポートのうち3つを使用しました」という警告なし。

## Photopea：Photoshopに驚くほど近い

[Photopea](https://www.photopea.com) はこのリストで最も技術的に印象的なツールであり、最も予想外でもあります。一人の開発者 — Ivan Kutskir氏がほぼ独力で — PSDファイル（Photoshopのネイティブ形式）、XCFファイル（GIMPの形式）、Sketchファイル、AIファイル（Adobe Illustrator）、PDF、SVG、その他の標準ウェブ形式を扱えるブラウザベースの画像エディタを構築しました。

ページを開くとすぐにエディタが表示されます。名前を尋ねるローディング画面なし、プラン選択なし、スキップするツアーなし。2010年頃のPhotoshopのような見た目ですが、それで構いません。2010年頃のPhotoshopは非常に機能的でした。

レイヤー、調整レイヤー、スマートオブジェクト、ブレンドモード、マスク、フィルター、カーブが利用できます。これらの機能の近似ではなく、実際のものです。PhotoshopがインストールされていないマシンでクライアントのPSDファイルをPhotopea で開き、タイプレイヤーとレイヤーエフェクトを維持したまま作業したことがあります。Canvaにはこれは全くできません。テンプレートツールであって、ファイルエディタではありません。

ビジネスモデルは無料ティアでの広告（約9ドル/月の有料広告なしプランがあり、Kutskir氏のソロ開発を支えています）。サーバーサイドでは何も処理されません — ファイルはブラウザに留まります。Photopea は月間アクティブユーザー約1000万人を報告しており、一人のプロジェクトとしては驚くべき数字です。[Kutskir氏の開発ブログ](https://blog.photopea.com) は、一人でこれほどの規模のものをどう維持しているかに興味があれば読む価値があります。

全機能の概要は [nologin.tools の Photopea 掲載ページ](/tool/photopea-com) をご覧ください。

**最適な用途**：レイヤーファイルの編集（PSD、XCF、AI）、写真レタッチ、合成、本格的なレイヤーコントロールが必要なもの。
**向かない用途**：テンプレートから簡単なSNSグラフィックを作る場合 — それはこのツールの目的ではありません。

## Excalidraw：意図的に手書き風の図表ツール

[Excalidraw](https://excalidraw.com) はすべてを手書き風に見せます。これは偶然ではありません。わずかに歪んだ形と手書きフォントは「下書き」を伝えます — これはシステムアーキテクチャ図、技術的なフローチャート、まだ検討中のワイヤーフレームにとってまさに適切なシグナルです。

ログイン不要はデザイン哲学に組み込まれています。リアルタイムコラボレーションは、参加者全員にアカウントが不要な共有URLで機能します。コラボレーションセッションはエンドツーエンドで暗号化されており、Excalidrawのサーバーでも描いている内容を見ることはできません。これは機密性の高い技術作業にとって重要なプライバシー特性です。

Excalidrawは [MIT ライセンスのオープンソース](https://github.com/excalidraw/excalidraw) で、GitHub スター数は85,000以上 — 存在する最もスター数の多いオープンソース描画ツールの一つです。Notion、GitLab、Linear、その他多数の製品に組み込まれています。このライブラリは広く使用されており、それを仕事で依存している人々によって維持・改善されています。

Canvaとの比較は示唆に富みます。Canvaで洗練された図表を持ってデザインレビューに臨むと、人々は何か最終版を提示していると思います。Excalidrawのスケッチで臨むと、会話はオープンなまま — 人々はそれに追加したり、疑問を呈したり、出発点として扱うことを気軽に感じます。どちらのモードも異なるプロジェクトステージで重要です。

[nologin.tools の Excalidraw フルプロフィール](/tool/excalidraw-com) に詳細があります。

**最適な用途**：技術図、システムアーキテクチャスケッチ、ワイヤーフレーム、「下書き」が意図したメッセージである視覚的ブレインストーミング。
**向かない用途**：印刷や公開される洗練されたアウトプットが必要な場合。

## tldraw：チームに実際に機能するホワイトボード

[tldraw](https://tldraw.com) は Excalidraw と Miro のようなものの中間に位置します。アウトプットはよりクリーンに見えます — 手書き風の美学はありません — しかし、従来のデザインツールではなく、無限キャンバスのホワイトボードのままです。形状はシャープで、UIは現代的で、Excalidrawと同様の方法でリアルタイムコラボレーションを処理します：URLを共有するだけで誰でもアカウント作成なしで参加できます。

tldrawが特に優れているのは空間的な視覚的思考です。付箋を追加したり、実際に見栄えの良い矢印を描いたり、リッチテキストを書いたり、画像を埋め込んだり、キャンバスのセクションをグループ化するフレームを作成したりできます。PNG と SVG にエクスポートできます。全体的な体験はレスポンシブで、2008年に設計されたように感じません。

開発者SDKは広く使用されています — [tldraw の GitHub リポジトリ](https://github.com/tldraw/tldraw)は約35,000スターで、`tldraw` npm パッケージはキャンバス体験を必要とする多くの製品に組み込まれています。つまり、基礎となるコードはエンドユーザーだけでなく、それを使って製品を構築している人々によって精査・改善されています。

2024年、tldraw はAI支援キャンバス機能のデモを披露しました — UIワイヤーフレームをスケッチして動作するHTMLを生成させるなど。これらの機能は実験的ですが、この方向性はtldrawがキャンバスをよりスマートにすることに投資していることを示唆しています。注目に値します。

[nologin.tools の tldraw 掲載ページ](/tool/tldraw-com) に詳細があります。

**最適な用途**：チームのブレインストーミングセッション、視覚的プランニング、サインアップの摩擦なしにコラボレーションする必要がある軽量ワイヤーフレーミング。
**向かない用途**：特定のファイル形式のアウトプットが必要なソロ作業。

## SVG-Edit：ブラウザでの本格的なベクター編集

[SVG-Edit](https://svgedit.netlify.app/editor/index.html) はこのリストで最も古いツールで、見た目もそれを反映しています。インターフェースは機能的で、ユーティリティナイフが機能的で時代を感じさせるのと同じ意味で時代を感じさせます — 美しく見せようとする試みなしに仕事をこなします。

SVG-Editのケースは明確です：ソフトウェアをインストールせずにSVGファイルを作成または編集する必要があり、クリーンなアウトプットが欲しい。SVG-Editは直接SVG形式に書き込みます — エクスポート時に内部表現が変換されることはありません。描いたものがファイルに入ります。

これはウェブ作業で重要です。Canvaは一部のプランでSVGをエクスポートできますが、CanvaのSVGアウトプットはごちゃごちゃしがちです：多くのネストされた`<g>`要素、生成されたID、冗長な属性。SVG-Editのアウトプットはそれに比べて最小限で、コードエディタで開いて手動で修正したり、オプティマイザーを通したりするのが容易です。

オープンソースで長年続いています — [SVG-Edit GitHub リポジトリ](https://github.com/SVG-Edit/svgedit) は複数の大規模な書き直しを経てメンテナンスされ続けています。コードベースは現在モジュラーESモジュールです。最も華やかなプロジェクトではありませんが、積極的にメンテナンスされており信頼できます。

[nologin.tools の SVG-Edit](/tool/svgedit-netlify-app-editor-index-html) に簡単な概要があります。

**最適な用途**：コードで直接使用するためのクリーンなSVGアイコン、ロゴ、イラストが必要なウェブ開発者。
**向かない用途**：フレンドリーなインターフェースが必要な場合、写真を扱う場合、または古いUIに耐性がない場合。

## Haikei：Canvaにできないことをするツール

[Haikei](https://haikei.app) は高度に特化されており、これが他のものと一緒に、ではなく代わりにお勧めする理由です。装飾的なSVGバックグラウンド、ブロブ形状、ウェーブ区切り、グラデーションメッシュ、レイヤードウェーブ、抽象パターンを生成します — 30以上のジェネレータータイプ — そしてクリーンな本番環境対応のSVGをエクスポートします。

ワークフローは：ジェネレーターを選んで、パラメーターを調整して、何かが良く見えるまでランダム化を繰り返し、エクスポートする。2分間。アウトプットはスケーラブルベクターグラフィックスで、どの画面サイズでも画素化なし、ファイルはHTMLに直接インライン化するほど小さいです。

Canvaにはいくつかの背景オプションがありますが、それらは固定サイズのビットマップであり、設定可能なベクターアセットではありません。ウェブサイトを構築していて、ブランドカラーに合いセクション間を滑らかにつながるウェーブ区切りが必要な場合、Haikeiはまさにそれを制御できます。Canvaの代替品ではなく、Canvaが残す空白を埋めるツールです。

コアジェネレーターはアカウントなしで無料です。Proティアにはより多くのジェネレータータイプが追加されます。

[nologin.tools の Haikei](/tool/haikei-app) で無料ティアに含まれるものを確認できます。

**最適な用途**：ランディングページやアプリインターフェース向けのバックグラウンド、ブロブ形状、ウェーブ区切りなどの装飾SVGアセットを作成するウェブ開発者とデザイナー。
**向かない用途**：特化したバックグラウンドジェネレーターではなく、汎用デザインツールが必要な場合。

## どれが自分に合っているか

「Canvaの代替品」というフレーミングはやや誤解を招きます。なぜならCanvaは主にテンプレートライブラリにエディタが付いたものだからです。Canvaを便利にしているのはエディタではなく — SNS投稿、プレゼンテーション、チラシ、名刺向けの何千もの出来合いのテンプレートです。これら5つのツールはどれもそれを再現していません。

代わりに、それぞれが特定の分野での本物の深みを持っています：

| ツール | 主な用途 | オープンソース | エクスポート形式 |
|------|-------------|-------------|----------------|
| [Photopea](https://www.photopea.com) | 写真編集、レイヤーファイル | いいえ | PSD, PNG, JPG, SVG, PDF |
| [Excalidraw](https://excalidraw.com) | 技術図、ワイヤーフレーム | はい (MIT) | PNG, SVG |
| [tldraw](https://tldraw.com) | コラボレーションホワイトボード | はい | PNG, SVG |
| SVG-Edit | クリーンなSVGベクター作成 | はい | SVG |
| [Haikei](https://haikei.app) | 装飾SVGアセット | いいえ | SVG, PNG |

いずれもサインアップ不要。いずれもサーバーでファイルを処理しません（tldrawのリアルタイムマルチプレイヤー同期を除く、これはその機能の要点です）。プラグインなしにモダンブラウザで動作します。

> プライバシー重視であることは偏執的であることを意味しません。単に、自分が残すデータについて意識的に選択することです。

これらのツールを組み合わせることで、特にCanvaのテンプレートエコシステムを必要としないほとんどのデザイン作業をカバーできます — そしてアカウント、デザイン履歴、管理するサブスクリプションを残さずにそれができます。[nologin.tools](/tool/nologin-tools) のより広いディレクトリには、多くのカテゴリにわたる同様のツールが掲載されており、コレクションは増え続けています。

ブラウザベース、プライバシー重視、サインアップなし — これは制約のように思えますが、今最も興味深いソフトウェアを生み出していることがわかります。あなたをロックインせずに生き残るツールは、実際に優れているものである傾向があります。

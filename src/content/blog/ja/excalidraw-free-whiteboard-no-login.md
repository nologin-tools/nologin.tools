ほとんどのホワイトボードツールは、最初の線を引く前にアカウント作成を求めてきます。Miro はメールアドレスが必要で、FigJam は Figma アカウントが必要、Lucidchart は5つ以上の図形を描こうとすると課金を求めてきます。さらに、有料プランを使えば、あなたが描いたものはすべて相手側からも見えてしまいます。

[Excalidraw](https://excalidraw.com) を開くと、すぐに空白のキャンバスが現れ、そのまま使い始めることができます。サインアップも不要、ログインも不要、メールアドレスを求めるポップアップも出てきません。ただのホワイトボードがそこにあるだけです。

これがこのツールの約束です。数百万人のユーザーと何年もの継続的な開発を経た今でも、その約束はしっかりと守られています。

## Excalidraw でできること

Excalidraw はブラウザベースの描画ツールで、すべての要素が手書き風のスタイルで描画されます。長方形の辺は少しだけ歪んでいて、線には自然なゆらぎがあります。完成度の高いスライドよりも、ナプキンに走り書きしたような雰囲気が漂います。

このビジュアルスタイルは意図的なデザインの選択であり、技術的な限界ではありません。アーキテクチャ図を描いたり、チームに概念を説明したりするとき、ラフな見た目は「これはドラフトなので、遠慮なくコメントしてください」というメッセージをキレイなプレゼンスライドよりもずっとうまく伝えてくれます。エンジニアリングチームはシステム設計の面接で使い、デザイナーは Figma を開く前のワイヤーフレーム段階で使い、教師はスライドの形式に縛られずに概念を説明するために使っています。

図形ツールは一般的なものが揃っています——長方形、円、矢印、線、テキスト、フリーハンド描画——さらにキャンバスに画像を直接埋め込むこともできます。オブジェクトのグループ化、レイヤー設定、ロック、グリッドへのアラインメントも可能で、カラーピッカー、線幅の調整、塗りつぶしオプションも備わっています。有料プランでしか使えない機能はありません。

エクスポートはアカウントなしで使えます。PNG（透過背景も選択可能）、SVG として保存したり、クリップボードにコピーしたりできます。`.excalidraw` ファイル形式はプレーンな JSON なので、アプリがなくても図面を読み取ることができます——プロジェクトがオフラインになった場合でも安心です。

## プライバシーを守る設計

Excalidraw をプライバシーに配慮したツールとして取り上げる理由はここにあります。コラボレーション機能がデフォルトでエンドツーエンド暗号化を採用しているのです。

「ライブコラボレーション」リンクで図面を共有する際、ルーム ID と暗号化キーの両方が URL のフラグメント（`#` より後ろの部分）に埋め込まれます。フラグメント部分はサーバーに送信されることなく、ブラウザ内に留まります。Excalidraw のサーバーは参加者間で図面データを中継しますが、受け取るのは暗号化されたバイト列だけで、描かれた内容を読み取ることはできません。

これは実質的なプライバシーの保証です。Miro や Notion のようなツールでは、プラットフォームがホワイトボードの内容に完全にアクセスできます。Excalidraw のコラボレーションモードでは、そうはなりません。[ソースコードは GitHub で公開](https://github.com/excalidraw/excalidraw)されており、MIT ライセンスのもとで誰でも確認できます——信頼を求めることなく、コードを読んで検証できます。

一人で使う場合は、データはブラウザの外に出ません。図面は `localStorage` に保存され、明示的にエクスポートしない限りデバイス上に留まります。

> "Your data is end-to-end encrypted, meaning the Excalidraw server cannot read what you've drawn." — Excalidraw documentation

これは、他のプライバシーを尊重するツールにも共通する設計思想です。サーバーはデータの輸送だけを担い、内容には触れない。鍵はユーザーが握っています。

## Excalidraw vs. tldraw vs. Diagrams.net

無料・ログイン不要のホワイトボードカテゴリには、いくつかの有力な選択肢があります。比較してみましょう：

| | Excalidraw | tldraw | Diagrams.net |
|---|---|---|---|
| ログイン必須 | いいえ | いいえ | いいえ |
| コラボレーション | E2E 暗号化 | あり | なし（エクスポートのみ） |
| スタイル | 手書き風 | シンプル／ベクター | 技術／UML |
| セルフホスト | 可能 | 可能 | 可能 |
| エクスポート形式 | PNG, SVG, JSON | PNG, SVG, JSON | PNG, SVG, PDF, XML |
| オープンソース | MIT | MIT | Apache 2.0 |

[tldraw](/tool/tldraw-com) は最も近い競合です。こちらも無料、ログイン不要、オープンソースです。主な違いはスタイルにあります——tldraw がきれいなベクター図形を使うのに対し、Excalidraw は徹底して手書きスタイルを貫いています。精度が求められる場合（正式な技術ドキュメントに入れる図など）は、tldraw のほうが扱いやすいでしょう。

[Diagrams.net](/tool/app-diagrams-net) は完全に異なるユースケースを対象としています。UML 図形、フローチャートのテンプレート、ネットワークトポロジーのアイコン、Confluence などの企業ツールと連携できる XML ベースの形式などを備えています。構造化された図の作成には強力ですが、自由なスケッチには向いていません。

Miro はエンタープライズ向けの選択肢です——洗練されていて機能も豊富ですが、無料枠が終わると1シートあたり月額16ドル以上かかります。アカウントが必要で、利用状況が追跡され、描いたものへのフルアクセスが相手側に渡ります。ときどき図を描くだけの小チームにとっては、Excalidraw が無料で提供してくれるもののために払う対価としては大きすぎます。

## データを渡さないコラボレーション

一般的なリアルタイムコラボレーションツールの仕組みはこうです——アカウントを作り、コンテンツはそのサーバー上に置かれ、相手側から読み取れる状態になります。これは既知のトレードオフで、多くのツールではそれで問題ありません。

Excalidraw のモデルは違います。2人がリアルタイムで同じキャンバスを編集できる——カーソルには名前が表示され、変更は即座に反映される——しかし中継サーバーはコンテンツの内容を実質的に知ることができません。暗号化キーがサーバーに触れることはないため、たとえ法的な要求があっても、読み取り可能な図面データは取り出せません。

コラボレーションセッションを始めるには、ツールバーの人物アイコンをクリックしてリンクを共有するだけです。リンクがあれば、アカウント作成なしで誰でも参加できます。セッションは誰かが接続している間だけ維持されます。無料プランには、永続的なクラウドルームはありません。

このため、バージョン履歴、デバイスをまたいだクラウド同期、ファイルをエクスポートせずに数日後に同じセッションに戻ることはできません。一時的なスケッチセッションであれば、これは許容できるトレードオフです。継続的なチームホワイトボードが必要な場合は、定期的に `.excalidraw` ファイルを共有フォルダにエクスポートするか、[Excalidraw+](https://plus.excalidraw.com)——永続的なクラウドストレージ、パスワード保護されたルーム、シーンのバックアップを追加した有料ホスト版——を検討するとよいでしょう。

無料版は、ほとんどの人が実際に必要とすること——同僚とホワイトボードでアイデアを整理してエクスポートし、次に進む——には十分に対応しています。

## キーボードショートカットと快適な操作感

Excalidraw のキーボードショートカットを覚えると、作業が格段に速くなります。指が覚えてしまえば、描くことに集中できます。

図形のショートカットはどれも1キーです。`R` で長方形、`E` で楕円、`A` で矢印、`L` で直線、`X` でフリーハンド、`T` でテキスト。`V` で選択ツールに戻ります。水平・垂直の反転も `H` と `V` で行えます。`Ctrl+A` で全選択、`Ctrl+G` で選択したオブジェクトをグループ化します。

共有操作：`Ctrl+Shift+E` でエクスポートダイアログを開き、クリップボードへのコピーやダウンロードができます。`Ctrl+L` で共有リンクを直接クリップボードにコピーできます。

ズーム操作：`Ctrl+スクロール` で拡大縮小、`Ctrl+Shift+H` で図面全体を画面に収めます。手のひらツール（`Space+ドラッグ`）は、選択中のツールを変えることなくキャンバスをパンできます。

これらのショートカットは、考えることと描くことの間の距離を最小限にしてくれます。アイデアが頭に浮かんでいるうちにメニューを探し回らずに済む、この即時性こそが Excalidraw を素早い図作成に向いていると思わせる主な理由です。

## オープンソースのエコシステム

Excalidraw の真の強みの一つは、その上に他の人たちが作り上げたものにあります。MIT ライセンスで、npm パッケージとして提供されているため、驚くほど多くのツールに組み込まれています。

[Excalideck](/tool/excalideck-com) は Excalidraw の図面をプレゼンテーションスライドに変換します——手書き風のスタイルをそのままスライド形式で、追加のソフトウェアは不要です。ツールを切り替えずに粗いアーキテクチャ図を見せたい技術系の発表に役立ちます。

Obsidian のプラグインでは、ノートの中に直接 Excalidraw の図を描くことができます。VS Code の拡張機能ではコードの横にキャンバスが埋め込まれます。いくつかのドキュメントツールや Wiki もExcalidraw 連携を追加しており、チームが図と説明テキストを同じ場所に置けるようにしています。

ライブラリ機能も注目に値します。コミュニティが数百種類の図形コレクションを提供しています——AWS アーキテクチャのアイコン、Google Cloud のインフラ図、モバイル UI コンポーネント、フローチャートのテンプレート、データベース図など。アプリ内のライブラリブラウザからインストールでき、登録は不要です。

プロジェクトは GitHub で 80,000 以上のスターを獲得しており、最も広く使われているオープンソースの描画ツールの一つです。長年の継続的な開発の結果として、活発なメンテナンス、迅速に対応される issue トラッカー、健全な連携エコシステムが実現しています。

## Excalidraw が向いている人

アーキテクチャ図を描くなら、Excalidraw はほぼ最良の選択です。手書きスタイルのおかげで、アイデアが固まる前から見栄えを気にする必要がなく、キーボードショートカットで思考と同じスピードで描くことができます。

UX ワイヤーフレームの lo-fi ステージ——Figma や Sketch を開く前の段階——でも Excalidraw は有効です。粗いスタイルは「これは探索段階です」と明確に伝え、ピクセルパーフェクトなモックアップよりも率直なフィードバックを引き出す傾向があります。

教育の現場では、先生がビデオ通話中に図を描き、生徒が共有キャンバスに参加するという使い方をしています。アカウント不要というモデルはここで特に重要です——すべての生徒がまた別のプラットフォームのアカウントを持っていたり、作りたいと思っていたりするとは限りません。

正確な位置揃え、条件付き書式、または正式な仕様書に入れる図が必要な場合は、[Diagrams.net](/tool/app-diagrams-net) や専用のベクターツールを使うべきです。手書きスタイルはExcalidraw の魅力であると同時に、その限界でもあります。

## セットアップ不要ですぐ始める

[excalidraw.com](https://excalidraw.com) にアクセスして、描き始めるだけです。オンボーディングはそれだけです。

作業中に図面が `localStorage` に自動保存されるため、タブを閉じて再び開いても前回のキャンバスが戻ってきます。長期的に保存したいものは `Ctrl+Shift+E` でエクスポートしましょう——`.excalidraw` ファイル（後で再度開いて編集できる）か PNG/SVG（共有や埋め込みに使う）として保存できます。

コラボレーションの場合は、人物アイコンをクリックしてルームリンクを共有するだけです。相手は何もインストールする必要はなく、アカウントもダウンロードも不要です。リンクだけあれば始められます。

自社のイントラネット、学校のネットワーク、あるいは図面の保存場所を完全に自分でコントロールしたいといった理由でセルフホストしたい場合は、Docker イメージが利用可能で、セルフホストのドキュメントも充実しています。MIT ライセンスなので、好きなように変更して実行できます。

アカウント作成を一切省いたツールキットを構築したい方は、[nologin.tools](/tool/nologin-tools) でさらに多くのプライバシーに配慮したログイン不要ツールを探してみてください。

ホワイトボードツールというカテゴリは、無料・オープンソース・プライバシーファーストの選択肢が、ほとんどのユーザーにとって本当に最良の選択肢となっている珍しい例です。そうなることは滅多にありません。Excalidraw がそれを実現できたのは、ユーザーのプライバシーとローカルでのデータ所有を後付けの機能としてではなく、製品の中心的な価値として扱ってきたからです。

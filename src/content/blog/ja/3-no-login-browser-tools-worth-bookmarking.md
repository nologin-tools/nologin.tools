本当に便利なツールは、自ら宣伝しないものです。Stack Overflow の回答にリンクとして紛れていたり、どこかのスレッドでさりげなく言及されていたり、同僚から「もう知ってるよね？」という前提でさっと教えてもらったりするものです。一度試してみると、まさに期待通りの動作をする。そしていつの間にか、自分の作業フローに欠かせない存在になっています。

今回紹介する 3 つのツールは、まさにそういう存在です。プライバシーに配慮し、ログイン不要で、余計な機能も派手な演出もありません。

## ExplainShell：ターミナルコマンドの「解読器」

`tar -xzf archive.tar.gz --strip-components=1 -C /usr/local` のようなシェルコマンドを見つけたとき、選択肢はざっくり 2 つです。そのまま実行して投稿者を信じるか、[ExplainShell](https://explainshell.com) に貼り付けるかです。

ExplainShell はシェルコマンドを解析し、各部分を対応する man ページの該当箇所に紐づけます。要約や言い換えはせず、各フラグ・各引数・各サブコマンドの公式ドキュメントをそのまま表示します。たとえば、先ほどの tar コマンドの `--strip-components=1` なら、ExplainShell は tar の man ページから直接引用しながら、「展開時にファイル名の先頭にあるパス要素を取り除く」という正確な意味を教えてくれます。

このツールは Idan Kamara が開発し、[GitHub](https://github.com/idank/explainshell) でオープンソースとして公開されています。仕組みがよくできていて、man ページをインデックス化したうえで、パーサーでコマンドを構成要素に分解し、それぞれを対応するドキュメントにリンクします。AI が推測するのではなく、正規ソースに対してマッチングを行っているわけです。

特に威力を発揮するのは、複数のツールを組み合わせた複雑なコマンドです。たとえば `find . -name "*.log" -mtime +30 -exec rm {} \;` には、`find`・位置指定テスト・`exec` 構文が同時に登場します。ExplainShell はそれぞれの要素をカラーコーディングで視覚的に分解してくれるので、一文字も読む前から全体の論理構造が見えてきます。

登録は不要です。コマンドを貼り付けてエンターを押せば、分解結果が表示される。それだけです。ドキュメントや同僚からシェルスクリプトを受け取ることが多い開発者にとって、週に何度も何気なく開くツールになるでしょう。そして、ブックマークのない端末を使う羽目になったとき、初めてその存在のありがたさを実感するはずです。

## PairDrop：ストレスなしのファイル転送

ファイル転送で一番困るのは、同じデスクにある 2 台のパソコン間でのやりとりではありません。400MB の動画を Android スマートフォンから Windows ノート PC に送るとき、2 台が別々のエコシステムに属している場合です。AirDrop は Apple デバイス間でしか使えない。Android の Nearby Share は Android 同士のみ。Bluetooth は遅い。USB-C ケーブルは物理的な接続と対応ポートが必要です。

[PairDrop](https://pairdrop.net) は、何もインストールせずにこの問題を解決します。モダンなブラウザさえあれば、どの 2 台のデバイスでも使えるブラウザベースのピアツーピア転送ツールです。スマートフォンとノート PC、2 台のノート PC、タブレットとデスクトップ——同じローカルネットワーク上で両方を開くだけで、自動的にお互いを検出します。片方をクリックし、もう一方で確認すれば、転送開始です。

ピアツーピアという点が重要です。ファイルはブラウザのビデオ通話でも使われる WebRTC を通じて、デバイス間で直接転送されます。PairDrop のサーバーは最初の接続確立だけを担当し、それ以降のデータ経路はデバイス間で完結します。機密文書や大きなファイルを転送する際、データがどこかのクラウドに保存されることを避けたいなら、WeTransfer や Google Drive とは根本的に異なるアプローチです。

PairDrop は [Snapdrop](https://snapdrop.net)——同コンセプトのオープンソースプロジェクト——をベースに大幅に機能強化したフォークです。Snapdrop になかった機能が追加されています。異なるネットワークのデバイスをつなぐパスワード保護付きルーム、テキスト転送、大きなファイルのより安定した処理などです。[GitHub](https://github.com/schlagmichdoch/PairDrop) でオープンソースとして維持されています。

[ShareDrop](/tool/sharedrop-io) との比較も触れておきましょう。どちらも登録不要のブラウザ P2P ファイル共有ツールです。ShareDrop は両デバイスが同一ネットワーク上にあることが前提。PairDrop はルーム機能によって異なるネットワーク間でも接続できます。自宅での利用ならどちらでも問題ありませんが、アカウントや容量制限を気にせず離れた人にファイルを送りたいなら、PairDrop に分があります。

ひとつ注意点として：WebRTC を使う性質上、企業のファイアウォールがピアツーピア接続をブロックするケースがあります。通常の自宅ネットワークやカフェの Wi-Fi では問題なく動作します。

## Markmap：ノートをマインドマップに変える

計画を立てているときの、あの瞬間があります。頭の中には構造がある——プロジェクトのアウトライン、調査の全体像、意思決定ツリー——でも、箇条書きの羅列ではそれを捉えきれなくなってきた。関係性を見たい。分岐を見たい。どのサブトピックがどこに属するのか、見たい。

そんなときに活躍するのが [Markmap](https://markmap.js.org) です。

Markmap は Markdown——具体的には Markdown の見出しとリスト——をインタラクティブなマインドマップに変換します。次のように書くと：

```markdown
# Project Launch

## Marketing
### Blog posts
### Social media
### Email campaign

## Engineering
### Backend API
### Frontend
### Testing

## Design
### Brand refresh
### Assets
```

クリックで折りたたんだり展開したりできる、インタラクティブな放射状ツリーとして描画されます。描画はリアルタイム。アップロードも不要、アカウントも不要、待ち時間もありません。左側にテキストエディタ、右側にマインドマップが表示され、入力に合わせてリアルタイムで更新されます。

エクスポート形式は実用的です。SVG と HTML に対応しています。SVG は、スライドデッキやドキュメントにそのまま貼り込めるベクター画像です。HTML は自己完結型のインタラクティブファイルで、ブラウザさえあれば受け取り手はアカウントなしで開けます。どちらの形式も、受け取る側に追加のインストールは一切必要ありません。

VS Code ユーザーには、Markdown ファイルの横にリアルタイムでマインドマップのプレビューを表示する [Markmap 拡張機能](https://marketplace.visualstudio.com/items?itemName=gera2ld.markmap-vscode)があります。それ以外のユーザーも、markmap.js.org のブラウザ版を使えばアカウントも支払いも「Pro へのアップグレード」の催促も一切ありません。

Markmap はフル機能のマインドマップアプリケーションを目指していません。ノードの手動配置、ノードごとのカラー設定、クラウド同期——これらの機能はありません。MindMeister、Miro、Coggle といったツールには揃っていますが、いずれも登録が必要です。レイアウトをピクセル単位でコントロールしたいなら、そちらを使えばよいでしょう。ただ、頭の中にある構造をすばやく外部化したいだけなら、Markmap の速さに敵うものはありません。

このプロジェクトはオープンソースで、[gera2ld が GitHub でメンテナンス](https://github.com/markmap/markmap)しており、開発者・技術ライターのコミュニティで広く支持されています。コアライブラリは npm パッケージとしても提供されており、ナレッジベースソフトウェアや静的サイトジェネレーターに Markmap のレンダリング機能を組み込む事例も多くあります。

## なぜこの 3 つなのか

「使うのをやめられないツール」と言うとき、当然の疑問が浮かびます——他のツールが定着しなかったなかで、なぜこれらは残ったのかということです。

ExplainShell が残った理由は、一つのことだけをやり、しかも一次情報源からやっているからです。シェルコマンドを「説明」するブラウザ拡張や AI チャットボットはいくらでもあります。でも ExplainShell は解釈をしているのではなく、実際のドキュメントをインデックスしているのです。それは精度の基準が高いということです。ターミナルで実行しようとしているものには、その高さが必要です。

PairDrop が残った理由は、クロスプラットフォームの P2P 転送というケースに対して、他にきれいに対処できるものがないからです。Apple 同士には AirDrop がある。Android 同士には Nearby Share がある。混在するケース——実際のファイル転送のほとんどは混在している——には、ネイティブで良いソリューションがありません。PairDrop はブラウザのタブ一枚で、登録なし、実質的な容量制限なしに、その空白を埋めてくれます。

Markmap が残った理由は、思考ツール（Markdown）とビジュアライゼーションツール（マインドマップ）の間の変換を、摩擦ゼロで実現しているからです。ほとんどのマインドマップソフトは、そのツールの流儀で考えることを強要します。Markmap は、Markdown で書いているならあなたのいる場所まで来てくれます。ツールがワークフローに合わせるのであって、ワークフローを強制するのではありません。

3 つともプライバシーの面でも優れています。データをローカルまたはピアツーピアで処理し、誰かのマーケティングデータベースに登録される心配がありません。また、時間が経つにつれて「フリーミアム」製品に変貌するタイプでもありません——ExplainShell は 10 年以上アカウント不要で無料、PairDrop はオープンソースで自己ホスト可能、Markmap のコアツールには有料の壁がありません。

このカテゴリのツールをもっと探しているなら、[nologin.tools](/tool/nologin-tools) が登録不要で使えるツールを検証・掲載したディレクトリを運営しています。デザイン系のツールについては[アカウント不要のデザインツール特集](/blog/five-design-tools-no-account)もあわせてご覧ください。

このパターンは確かに魅力的です——すぐに使えて、何も要求してこないソフトウェア。実はかなりの数があります。難しいのはこれらを見つけることではなく、デスクトップアプリを開いたりまた別のアカウントを作ったりする前に、まずそちらに手を伸ばす習慣をつくることです。一度その習慣がつくと、元には戻れません。

"隐私友好"这个词出现在你遇到的大约一半的网络工具上。没有一个能在没有你永远不会完整阅读的隐私政策的情况下证明这一点。开源改变了这一切。当源代码是公开的，工具在你的浏览器中运行时，这个声明就变得可审计了。这不是一个微小的区别。

关于大多数工具背后的登录墙，有一件事值得说清楚：它很少是关于安全的。它是关于数据的。你的电子邮件地址有价值。你的使用模式有价值。构建这些工具的公司有商业模式，即使工具本身是免费的，你也要用行为数据来支付——这些数据被打包用于广告、出售或用来训练 AI 模型。注册表单是这笔交易开始的地方。

开源无需登录工具完全颠覆了这一切。没有账号，没有服务器发送你的数据，而且——至关重要的是——代码是你或你信任的人实际上可以阅读的，以验证没有任何不正当的事情发生。

## 为什么"开源"是无需登录等式的另一半

跳过注册表单是必要的，但还不够。不需要登录的闭源工具仍然可以发出网络请求来记录你的活动、对你的浏览器进行指纹识别或窃取你的文件。你无法验证它不会这样做。

开源意味着代码是公开可用的（通常在 GitHub 上），任何人都可以检查它、审计它，或者在某些内容发生变化时报告。出于隐私目的，[开源倡议的定义](https://opensource.org/osd)要求源代码可以自由获取、可再发行和可修改——这意味着如果工具曾经添加跟踪功能，有人会注意到并提出问题。

真正重要的组合：开源 + 客户端处理 + 无需登录。移除其中任何一个，你就回到了相信营销文案的境地。

> "只要有足够多的眼睛，所有的 bug 都会浮出水面。" —— Eric S. Raymond，《大教堂与集市》。同样的原则适用于隐私侵犯。公开的代码以闭源代码永远无法做到的方式受到审查。

在 [nologin.tools](/tool/nologin-tools) 上精选的工具正是针对这一点进行检查的——客户端处理、没有隐藏的网络请求、没有注册墙。开源的工具更进一步，因为它们的架构是可以公开验证的。

## 可以阅读所运行代码的开发者工具

[Excalidraw](https://excalidraw.com) 可能是现存最广泛使用的开源无需登录工具。[GitHub 仓库](https://github.com/excalidraw/excalidraw)有超过 90,000 个 star。你打开它，画一个图表，你的数据保留在浏览器中（本地存储）。实时协作是可选的，基于会话——默认情况下没有人把你的草图存储在服务器上。查看 [Excalidraw 列表](/tool/excalidraw-com) 了解完整功能摘要，但这里的重点是架构：客户端优先的设计，而不是作为营销事后想法。

[Hoppscotch](/tool/hoppscotch-io) 是 Postman 的无需登录替代品。无需创建账号即可测试 REST、GraphQL、WebSocket 和 SSE 端点。它是开源的（MIT 许可证），在浏览器中处理所有内容——你的 API 请求直接从你的浏览器发送到目标端点，而不是通过 Hoppscotch 的服务器。当你测试内部 API 或使用你不想通过第三方路由的凭据时，这一点很重要。Postman 越来越强硬地要求账号；Hoppscotch 是干净的替代品。

还有 [IT Tools](/tool/it-tools-tech)——一个建立在开源代码上的 70 多种开发者工具集合（哈希生成器、UUID 工具、日期转换器、正则表达式测试器、颜色选择器）。所有内容都在客户端运行。整个项目在 GitHub 上，可以自托管。对于你经常使用的工具，这种类型的工具无论如何都值得在本地运行。

| 工具 | 许可证 | 可自托管 | 处理方式 |
|------|---------|---------------|------------|
| Excalidraw | MIT | 是 | 客户端 |
| Hoppscotch | MIT | 是 | 客户端 |
| IT Tools | MIT | 是 | 客户端 |
| CyberChef | Apache 2.0 | 是 | 客户端 |
| Mermaid Live | MIT | 是 | 客户端 |

[Mermaid Live Editor](/tool/mermaid-live) 将文本转换为流程图、序列图和甘特图——全在浏览器中，无需登录，开源。来自 Godbolt 的 [Compiler Explorer](/tool/godbolt-org) 可以在不注册的情况下显示 C++、Rust、Go 和数十种其他语言的汇编输出。这两个工具都是你作为开发者经常使用的工具，你永远不希望它们被账号门控。

## 代码与画布一样开放的创意工具

[Squoosh](https://squoosh.app) 是一个 Google 构建的图像压缩工具，完全开源，通过 WebAssembly 在你的浏览器中完成所有处理。你的图像永远不会离开你的设备。你可以压缩 PNG、JPEG、WebP 和 AVIF，并进行并排质量比较。Google 部分地将 Squoosh 作为 WebAssembly 展示案例构建——这意味着代码写得非常好，工具真的很快。没有注册，没有文件上传到服务器，没有超出浏览器处理能力的大小限制。[Squoosh 列表](/tool/squoosh-app) 有更多详情，但简而言之：这就是客户端图像处理的正确做法。

[SVGOMG](/tool/jakearchibald-github-io-svgomg) 由 Jake Archibald（前 Google Chrome 团队开发者倡导者）构建。这是 SVGO（SVG 优化库）的可视化界面。粘贴或上传 SVG，切换要应用的优化，然后下载结果。纯客户端。开源。这种工具只做一件事，而且做得很好，没有任何要求你输入电子邮件的表单字段。

[tldraw](/tool/tldraw-com) 值得在这里提一下——类似于 Excalidraw，但具有更强大的无限画布模型和不同的美学风格。开源，基本使用无需登录，数据默认保留在浏览器中。如果 Excalidraw 的手绘风格不适合你，tldraw 是另一个选择。

## 自我审计的安全工具

关于要求你在检查隐私之前先创建账号的隐私和安全工具，有一种近乎讽刺意味的东西。[PrivacyTests.org](/tool/privacytests-org) 不会这样做。这是一个开源项目，对主要浏览器运行自动测试，以检查跟踪保护、指纹识别抵抗和数据泄漏。测试方法是公开的，代码在 GitHub 上，你可以看到你的浏览器表现如何，而不需要给任何人你的电子邮件。

[CyberChef](/tool/gchq-github-io-cyberchef)——"网络瑞士军刀"——来自英国情报机构 GCHQ，这对于一个注重隐私的工具来说是一个不寻常的来源。但 CyberChef 是完全客户端的，开源的（Apache 2.0）。它可以在不离开浏览器的情况下编码、解码、加密、解密、分析数据并在格式之间转换。GCHQ 将其作为公共工具发布以帮助分析师，它已成为安全研究人员和 CTF 参与者的标准工具。它是开源的，这意味着独立研究人员已经验证了它的声明——如果你不想使用 GCHQ 托管的版本，你可以自托管它。

[PairDrop](/tool/pairdrop-net) 使用 WebRTC 进行点对点文件传输。你的文件直接发送到接收方的设备，不经过 PairDrop 的服务器。开源，无需登录，适用于任何具有现代浏览器的设备。服务器只促进对等发现——一旦连接，传输就是直接的。将其与将文件上传到服务器的文件传输服务进行比较：使用 PairDrop，没有什么可以存储的，没有什么可以被泄露的，验证这一点的代码是公开的。[ShareDrop](/tool/sharedrop-io) 以相同的方式工作，值得收藏作为备用。

## "无需注册"实际上给你带来了什么（技术层面）

[EFF 的网络隐私指南](https://www.eff.org/issues/privacy)提出了一个值得直接说明的结构性观点：即使是声称不出售你数据的工具也可以在内部使用它、与合作伙伴共享它，或者在数据泄露中丢失它。不收集数据的工具不会滥用它。没有登录表单不仅仅是方便——这是关于工具需要什么数据才能运行的架构声明。

对于开源无需登录工具，你可以直接验证这一点。克隆仓库，读取网络请求，在本地运行。这些工具中的大多数都是专门设计的，使自托管变得简单——[IT Tools](https://github.com/CorentinTh/it-tools) 的 README 有一个三行的 Docker 部署说明。[Hoppscotch](https://github.com/hoppscotch/hoppscotch) 有自托管指南。Excalidraw 可以部署到任何静态主机。

对大多数人来说，自托管不是必要的。但它*可能*这一事实使这些工具值得信赖。一个你可以自己运行但为了方便选择不运行的工具，与一个你别无选择只能使用其托管版本的工具，从根本上是不同的隐私情况。

## 值得关注的趋势

随着 WebAssembly 的发展，开源客户端工具的趋势加速了。[Mozilla 关于 WebAssembly 成为一流网络语言的文章](https://hacks.mozilla.org/2026/02/making-webassembly-a-first-class-language-on-the-web/)解释了为什么基于浏览器的工具现在可以匹配桌面应用程序的性能——这使得"我们需要服务器来处理这个"的借口越来越站不住脚。

图像压缩？在浏览器中运行（Squoosh）。图表创建？在浏览器中运行（Excalidraw、tldraw）。API 测试？在浏览器中运行（Hoppscotch）。代码编译和执行？在浏览器中运行（Compiler Explorer、Rust Playground、Go Playground）。真正需要服务器端处理的任务类别不断缩小。

当一个工具仍然坚持它需要在服务器上保存你的数据时——而且这不是像云同步那样本身就是整个要点的东西——问问为什么。有时有合理的技术原因。通常没有。

将持续存在的无需注册工具是那些架构使数据收集在结构上不可能的工具，而不仅仅是政策上禁止的。开源使该架构可验证。这种组合——开源、客户端、无需登录——是网络工具可以提供的最强隐私保证。

浏览 [nologin.tools](/tool/nologin-tools) 上的开源工具，在每个类别中找到经过验证的选项，从开发者工具到创意工具到隐私检查器。验证过程检查这里描述的确切属性。
